New Site — Functionality Audit & UX Opportunities

The 3-skin slot design is ported and live, and it looks fantastic. This is an honest pass on what works, what only looks like a button, whether the mouse/depth interactivity responds, and — the big one — what the simplification traded away versus the old dashboard, so we can decide what depth to earn back.

No changes made to the live site — findings only, per your ask.

I clicked and hovered these on the real deployed page.

3What the simplification traded away

The old dashboard was a deep tool; the new one is a beautiful glance. That's a good trade for a wall display — but here's the depth that got dropped, so we choose what to re-earn.

CRITICAL
In-app Approve / Pre-arm / Skip — the real-money control we just got working is gone from the display. (Backed up at /legacy-dashboard.html; Telegram still works.)
HIGH
Tap-a-trade / position for the "why" — the engine's reasoning per trade/holding is no longer reachable. It was a signature feature.
HIGH
"How it works" explainer + (i) icon — a friend glancing at the wall has no way to learn what they're seeing.
MED
Track Record scorecard — return, drawdown, and alpha vs buy-and-hold TQQQ (the honest scoreboard).
MED
Serious ⇄ Casual voice toggle — the two-voice engine-notes we just built aren't surfaced here.
MED
Paper-account view — watching the always-on autonomous testbed rip.
LOW
Catalyst-orbits map & full coverage intel, and the "how is that possible?" creed — nice-to-haves.

4Recommendations — prioritized

CRITICAL
Fold approvals into the designWhen the engine has a pending trade, slide in a per-skin approval card (Approve / Pre-arm / Skip + PIN) near the top. This is the one true functional gap — the site should let you act, not just watch.
HIGH
Default → Glass, and a real "vibe" toggle menuYour call, captured: make Glass the landing skin, and upgrade the toggle to a labeled menu (a word + one-line vibe each: "Glass — pro terminal / Neon — arcade / Gold — high roller") instead of three bare chips.
HIGH
Make positions & trades tappable → the "why"Tap a row to expand the engine's reasoning (we already store thesis per position and the note text per trade). Brings the depth back without cluttering the glance.
HIGH
Discreet (i) → the how-it-works pageOne small corner icon so anyone can learn what the machine is doing. The explainer page already exists.
MED
A small Track-Record / vs-TQQQ statOne honest line ("+1.1% all-time · vs +X% buy-hold TQQQ") — keeps the scoreboard visible.
MED
Surface the Serious ⇄ Casual voiceA tiny toggle on the notes panel — the dual-voice engine copy is already flowing.
POLISH
Fix false-affordances + Neon hero brightness + mobile passMake the pill/rows read correctly; nudge Neon's hero lighting to match Glass's readability; confirm the 1080→viewport scale reads well on the phone.

5Suggested sequence

1. Approvals card  →  2. Glass default + vibe menu  →  3. Tap-for-why + (i) explainer  →  4. Track-record stat + voice toggle  →  5. Polish (affordances, Neon light, mobile).

The through-line: keep the gorgeous glance, re-earn the depth. The new design is a better first impression than we've ever had — it just needs a few of the old tool's powers folded back in without losing the calm.
